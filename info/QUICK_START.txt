🚀 SETUP IN 3 STEPS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

STEP 1: Create Database
────────────────────────
$ mysql -u root -p < database_schema.sql

This creates:
  • nutricheck database
  • 5 tables (users, products, categories, nutrition_data, product_ratings)
  • Sample data (1 admin, 5 categories, 5 products)

STEP 2: Configure ColdFusion Datasource
────────────────────────────────────────
1. Open: http://localhost:8501/CFIDE/administrator/
2. Go to: Data & Services > Datasources
3. Add datasource:
   Name: nutricheck
   Database: nutricheck
   Server: localhost
   Port: 3306
   Username: root
   Password: [your MySQL password]
4. Click Submit and verify connection

STEP 3: Deploy Application
───────────────────────────
Copy nutricheck folder to ColdFusion webroot
(Usually: /opt/coldfusion/cfusion/wwwroot/ or similar)

🔧 TROUBLESHOOTING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Problem: Cannot connect to database
Solution: 
  • Check MySQL is running
  • Verify datasource in CF Admin
  • Check credentials

Problem: 404 Error
Solution:
  • Verify application path
  • Check webroot location
  • Restart ColdFusion

Problem: Login fails
Solution:
  • Clear browser cache
  • Verify admin user in database
  • Check session management

Problem: No products showing
Solution:
  • Verify sample data inserted
  • Check browser console
  • Test API endpoints
